## Target Candidate and Requirements

### Target Candidate Description

The target candidate should have:

- **2+ years of experience** building production-grade applications on AWS or with open-source technologies
- **General AI/ML** or data engineering experience
- **1+ year of hands-on experience** implementing GenAI solutions

### Recommended AWS Knowledge

The target candidate should have the following AWS knowledge:

| Area | Description |
|------|-------------|
| Compute, Storage, Networking | Experience with EC2, S3, VPC, Lambda services |
| Security | Understanding of AWS security best practices and identity management (IAM) |
| Deployment & IaC | Experience with CloudFormation, CDK, CodePipeline |
| Monitoring | Familiarity with CloudWatch, X-Ray and observability services |
| Cost Optimization | Understanding of AWS cost optimization principles |

### Job Tasks OUT OF SCOPE

The following tasks are **not** within the scope of the exam:

- **Model development and training** — The exam does not require you to know how to train models from scratch
- **Advanced ML techniques** — No deep understanding of advanced ML techniques like gradient descent, backpropagation
- **Data engineering and feature engineering** — No requirement to build complex data pipelines or feature engineering

{{% notice tip %}}
**Exam Tip:** The exam focuses on **using** and **integrating** Foundation Models, not **building** them. Focus on Amazon Bedrock, Knowledge Bases, Guardrails, and how to integrate FMs into real-world applications.
{{% /notice %}}

{{% notice warning %}}
Don't spend too much time studying model training or ML algorithms. Instead, focus on AWS services related to GenAI and how to use them in production.
{{% /notice %}}
